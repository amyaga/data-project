cd "C:\Users\amyag\Desktop\Data Project" 
    - переходим в папку, которую сделаем локальным репозиторием
git init 
    - включаем Git на эту папку
git remote add origin "https://github.com/amyaga/data-project.git"
    - связываем локальную папку с удалённым репозиторием
git status 
    - показать какие файлы изменились в локальном репозитории
git add.
    - добавить в индекс все изменённые файлы из локального репозитория
git commit -m "Add new files from local folder"
    - добавить в удалённый репозиторий все изменённые файлы
git log
    - вывести всю историю коммитов и их хеши
git checkout (хеш)
    - откатиться к версии, хеш которой указан
git push origin master
    - запушить все локальные коммиты в удалённый репозиторий (ветка master)
git pull
    - подтянуть все файлы из удалённого репозитория
git branch
    - показать список веток доступных в локальном репозитории
git branch feature
    - создание ветки feature для дополнительных изменений
git checkout feature
    - переключение на ветку feature
git branch --delete feature
    - удаляем ветку feature
git push origin feature
    - находясь в ветке feature, создаём в удалённом репозитории ветку с точно таким же именем

Для того, чтобы слить две ветки (merge feature и master)
git checkout master
    - переходим на ветку master (в которой хотим сделать merge)
git merge feature
    - делаем слияние из указанной ветки в ту, в которой сейчас находимся
git rebase feature
    - делаем слияние из указанной ветки в ту, в которой сейчас находимся и переносим вместе с ней историю всех коммитов
